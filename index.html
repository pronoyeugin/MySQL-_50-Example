Important SQL Commands

01. SELECT

extracts data from a database

SQL Statement: SELECT CustomerName, City, Country FROM Customers;

02. UPDATE

updates data in a database

SQL Statement: SELECT CustomerName, City, Country FROM Customers;

03. DELETE

deletes data from a database

04. INSERT INTO

inserts new data into a database

05. CREATE DATABASE

creates a new database

07. ALTER DATABASE

modifies a database

08. CREATE TABLE

creates a new table

09. ALTER TABLE

modifies a table

10. DROP TABLE

deletes a table

11. CREATE INDEX

creates an index (search key)

12. DROP INDEX

deletes an index

MySQL Exampale

01. SELECT

The following SQL statement selects the "CustomerName", "City", and "Country" columns from the "Customers" table:

SQL Statement: SELECT CustomerName, City, Country FROM Customers;

02. SELECT DISTINCT

The SELECT DISTINCT statement is used to return only distinct (different) values. Inside a table, a column often contains many duplicate values; and sometimes you only want to list the different (distinct) values.

SQL Statement: SELECT DISTINCT column1, column2, ... FROM table_name;

03. WHERE

It is used to extract only those records that fulfill a specified condition.

SQL Statement: SELECT * FROM Customers WHERE Country = 'Mexico';

04. AND

The AND operator displays a record if all the conditions separated by AND are TRUE.

SQL Statement: SELECT column1, column2, ... FROM table_name WHERE condition1 AND condition2 AND condition3 ...;

05. OR

The OR operator displays a record if any of the conditions separated by OR is TRUE.

SQL Statement: SELECT column1, column2, ... FROM table_name WHERE condition1 OR condition2 OR condition3 ...;

06. NOT

The NOT operator displays a record if the condition(s) is NOT TRUE.

SQL Statement: SELECT column1, column2, ... FROM table_name WHERE NOT condition;

07. Combining AND, OR and NOT

The following SQL statement selects all fields from "Customers" where country is "Germany" AND city must be "Berlin" OR "Stuttgart" (use parenthesis to form complex expressions):

SQL Statement: SELECT * FROM Customers WHERE Co untry = 'Germany' AND (City = 'Berlin' OR City = 'Stuttgart');

08. ORDER BY

The ORDER BY keyword is used to sort the result-set in ascending or descending order.

SQL Statement: SELECT column1, column2, ... FROM table_name ORDER BY column1, column2, ...;

09. INSERT INTO

he INSERT INTO statement is used to insert new records in a table.

SQL Statement: INSERT INTO table_name (column1, column2, column3, ...) VALUES (value1, value2, value3, ...);

10. NULL

If a field in a table is optional, it is possible to insert a new record or update a record without adding a value to this field. Then, the field will be saved with a NULL value.

SQL Statement: (Is NULL: SELECT column_names FROM table_name WHERE column_name IS NULL;) (Is NOT NULL: SELECT column_names FROM table_name WHERE column_name IS NOT NULL;)

11. UPDATE

The UPDATE statement is used to modify the existing records in a table.

SQL Statement: UPDATE table_name SET column1 = value1, column2 = value2, ... WHERE condition;

12. DELETE

The DELETE statement is used to delete existing records in a table.

SQL Statement: DELETE FROM table_name WHERE condition;

13. LIMIT

The LIMIT clause is useful on large tables with thousands of records. Returning a large number of records can impact performance.

SQL Statement: SELECT column_name(s) FROM table_name WHERE condition LIMIT number;

14. MIN()

The MIN() function returns the smallest value of the selected column.

SQL Statement: SELECT MIN(column_name) FROM table_name WHERE condition;

15. MAX()

The MAX() function returns the largest value of the selected column.

SQL Statement: SELECT MAX(column_name) FROM table_name WHERE condition;

16. COUNT()

The COUNT() function returns the number of rows that matches a specified criterion.

SQL Statement: SELECT COUNT(column_name) FROM table_name WHERE condition;

17. AVG()

The AVG() function returns the average value of a numeric column.

SQL Statement: SELECT AVG(column_name) FROM table_name WHERE condition;

18. SUM()

The SUM() function returns the total sum of a numeric column

SQL Statement: SELECT SUM(column_name) FROM table_name WHERE condition;

19. LIKE

The LIKE operator is used in a WHERE clause to search for a specified pattern in a column. There are two wildcards often used in conjunction with the LIKE operator. (%), (_)

SQL Statement: SELECT * FROM Customers WHERE CustomerName LIKE '%or%';

20. IN

The IN operator allows you to specify multiple values in a WHERE clause.

SQL Statement: SELECT column_name(s) FROM table_name WHERE column_name IN (value1, value2, ...);

21. BETWEEN

The BETWEEN operator selects values within a given range. The values can be numbers, text, or dates.

SQL Statement: SELECT * FROM Products WHERE Price BETWEEN 10 AND 20;

22. Aliases

Aliases are used to give a table, or a column in a table, a temporary name

SQL Statement: SELECT CustomerID AS ID, CustomerName AS Customer FROM Customers;

23. Joins

A JOIN clause is used to combine rows from two or more tables, based on a related column between them.

SQL Statement: SELECT Orders.OrderID, Customers.CustomerName, Orders.OrderDate FROM Orders INNER JOIN Customers ON Orders.CustomerID=Customers.CustomerID;

24. INNER JOIN

The INNER JOIN keyword selects records that have matching values in both tables.

SQL Statement: SELECT column_name(s) FROM table1 INNER JOIN table2 ON table1.column_name = table2.column_name;

25. LEFT JOIN

The LEFT JOIN keyword returns all records from the left table (table1), and the matching records (if any) from the right table (table2).

SQL Statement: SELECT column_name(s) FROM table1 LEFT JOIN table2 ON table1.column_name = table2.column_name;

26. RIGHT JOIN

The RIGHT JOIN keyword returns all records from the right table (table2), and the matching records (if any) from the left table (table1).

SQL Statement: SELECT Orders.OrderID, Employees.LastName, Employees.FirstName FROM Orders RIGHT JOIN Employees ON Orders.EmployeeID = Employees.EmployeeID ORDER BY Orders.OrderID;

27. CROSS JOIN

The CROSS JOIN keyword returns all records from both tables (table1 and table2).

SQL Statement: SELECT Customers.CustomerName, Orders.OrderID FROM Customers CROSS JOIN Orders;

28. Self Join

A self join is a regular join, but the table is joined with itself.

SQL Statement: SELECT A.CustomerName AS CustomerName1, B.CustomerName AS CustomerName2, A.City FROM Customers A, Customers B WHERE A.CustomerID <> B.CustomerID AND A.City = B.City ORDER BY A.City;

29. UNION

The UNION operator is used to combine the result-set of two or more SELECT statements.

SQL Statement: SELECT City FROM Customers UNION SELECT City FROM Suppliers ORDER BY City;

30. GROUP BY

The GROUP BY statement is often used with aggregate functions (COUNT(), MAX(), MIN(), SUM(), AVG()) to group the result-set by one or more columns.

SQL Statement: SELECT column_name(s) FROM table_name WHERE condition GROUP BY column_name(s) ORDER BY column_name(s);

31. HAVING

The HAVING clause was added to SQL because the WHERE keyword cannot be used with aggregate functions.

SQL Statement: SELECT COUNT(CustomerID), Country FROM Customers GROUP BY Country HAVING COUNT(CustomerID) > 5;

32. EXISTS

The EXISTS operator is used to test for the existence of any record in a subquery.

SQL Statement: SELECT column_name(s) FROM table_name WHERE EXISTS (SELECT column_name FROM table_name WHERE condition);

33. ANY

ANY means that the condition will be true if the operation is true for any of the values in the range

SQL Statement: SELECT column_name(s) FROM table_name WHERE column_name operator ANY (SELECT column_name FROM table_name WHERE condition);

34. ALL

ALL means that the condition will be true only if the operation is true for all values in the range.

SQL Statement: SELECT column_name(s) FROM table_name WHERE column_name operator ALL (SELECT column_name FROM table_name WHERE condition);

35. INSERT INTO SELECT

The INSERT INTO SELECT statement copies data from one table and inserts it into another table.

SQL Statement: INSERT INTO table2 (column1, column2, column3, ...) SELECT column1, column2, column3, ... FROM table1 WHERE condition;

36. CASE

The CASE statement goes through conditions and returns a value when the first condition is met (like an if-then-else statement)

SQL Statement: SELECT OrderID, Quantity, CASE WHEN Quantity > 30 THEN 'The quantity is greater than 30' WHEN Quantity = 30 THEN 'The quantity is 30' ELSE 'The quantity is under 30' END AS QuantityText FROM OrderDetails;

36. IFNULL()

The MySQL IFNULL() function lets you return an alternative value if an expression is NULL.

SQL Statement: SELECT ProductName, UnitPrice * (UnitsInStock + IFNULL(UnitsOnOrder, 0)) FROM Products;

37. COALESCE()

SQL Statement: SELECT ProductName, UnitPrice * (UnitsInStock + COALESCE(UnitsOnOrder, 0)) FROM Products;

37. Comments

Comments are used to explain sections of SQL statements, or to prevent execution of SQL statements.

SQL Statement: -- Select all: SELECT * FROM Customers;

38. Sigle line Comments

Single line comments start with --

SQL Statement: SELECT * FROM Customers -- WHERE City='Berlin';

38. Multi-line Comments

Multi-line comments start with /* and end with */.

SQL Statement: /*Select all the columns of all the records in the Customers table:*/ SELECT * FROM Customers;
